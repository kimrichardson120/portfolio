Commenting Feature

Improving user work flows through an easy-to-use commenting functionality

How might we help our users communicate important information to each other about specific purchase orders?

Project Details

CONTRIBUTION

  • Discovery
  • UX Research
  • UX Design
  • UI Design
  • User Testing

TOOLS USED

  • Mural
  • Figma

Challenge

The first product on which I lead design on in my current role is a receipt reconciliation application used by accountants. This application was created to offer a way for accountants to research receipts and purchase orders (POs) that were out of balance and needed to be reconciled. Since the receipts and POs are being reported at an enterprise level, there are a large quantity of items that need to be reconciled and multiple accountants work together to help each other research and reconcile these receipts and POs. The accountants needed a way better way to communicate with each other about specific receipts and/or POs to work more efficiently and create labor savings.

Research

I collaborated with the User Research team to create a research plan, including a competitive analysis, discussion guide and moderated testing plan. After conducting the competitive analysis I presented my findings and recommendations to stakeholders.

COMPETITIVE ANALYSIS OBJECTIVES

  • To explore commenting features in other products
  • To review commenting functionality in other products
  • To evaluate the key strengths and weaknesses in commenting features

RECOMMENDATIONS

  • Each comment displays the user who created the comment
  • Each comment has a timestamp with the date and time
  • Each comment is separated by a line
  • A comment icon is displayed in each line item field to indicate where a user can add or view a comment
  • A modal opens or a panel slides over to view or add comments
  • A comment icon is displayed in each line item field to indicate where a user can add or view a comment

User Flows

This is the user flow for a user who is viewing and/or adding comments.

Low Fidelity Wireframes

After conducting the competitive analysis and collaborating with my product team, I sketched some potential designs for how the commenting feature would work in our product.

Solution

The design that I landed on was to create a sliding panel that would move in from the right once the user has clicked the comment icon. The user can view and/or add comments in the sliding panel. There is also a record of who made each comment and when the comment was made. As this product is an application used for accounting, we did not give the user the ability to delete or edit comments as an audit trail is very important in accounting practices. I conducted user testing with 5 accountants to confirm that the user experience was effective. Once I analyzed the results of the user testing I collaborated with the Kroger design system team to get approval for this new user flow because no other internal Kroger application had a robust commenting feature.

Since this feature has been in production, we have made an iteration on where the comment icon is placed so that it is more clearly visible to the user if the table has a horizontal scroll on it. As this product is an internal accounting tool, I have not included images of the high-fidelity prototypes or the production application.

Impact

The commenting feature in the reconciliation application has improved the work efficiency of the accountants who use the application. The users now can comment directly on an item during their research; in the past the comments lived in Teams channels, ExCel sheets, emails, etc. The labor savings has been estimated at 5 minutes per receipt/PO and as most accountants work multiple POs per hour, this labor savings quickly adds up.

Another impact this feature has is that it is the first commenting flow used in an internal Kroger application comprised of components from the Kroger Design System. I worked with the design system team to approve the design and it is included in the Supply Chain, Manufacturing, Finance and Accounting (SCMFA) pillar design library roadmap to be implemented in the SCMFA design system.
